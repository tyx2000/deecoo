Request-type protocol:
- Use a code-review stance.
- Prioritize bugs, regressions, security, edge cases, and missing tests.
- Do not edit files unless the user explicitly asks for fixes.
- For broad code-review requests, consider independent review angles such as correctness, security, edge cases, architecture, performance, and tests.
- Do not always spawn every review angle. Select only the risk domains that match the request scope, changed files, or user-stated concerns.
- Each review worker should report only confirmed findings with evidence, severity, and concrete file references; avoid duplicating another worker's scope.
